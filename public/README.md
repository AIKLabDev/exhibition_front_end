# public 폴더

이 폴더 안의 파일은 **빌드 시 그대로** `dist/` 루트로 복사됩니다.  
Vite가 내용을 변경하거나 해시를 붙이지 않고, **경로 구조만 유지한 채** 그대로 내보냅니다.

---

## Vite에서 정적 파일이 처리되는 방식

**Vite**는 프론트엔드 빌드 도구로, 소스 코드와 리소스를 묶어서 `dist/`에 결과물을 만듭니다.  
정적 파일(이미지, 사운드 등)은 **어디에 두고 어떻게 참조하느냐**에 따라 처리 방식이 달라집니다.

### 1. 모듈 그래프에 들어가는 파일 (import)

- **위치**: 프로젝트 루트 아래 아무 경로 (이 프로젝트에서는 `resources/` 사용).
- **참조**: `import logoUrl from './resources/logo.png'` 처럼 **소스 코드에서 import**.
- **Vite 동작**:
  - 해당 파일을 **의존성**으로 인식하고 빌드 파이프라인에 포함합니다.
  - 이미지는 최적화·해시 부여 등이 적용될 수 있고, 결과는 보통 `dist/assets/` 아래에 `[이름]-[해시].[확장자]` 형태로 나갑니다.
  - import 한 변수(`logoUrl`)에는 **최종 URL 문자열**이 들어갑니다 (예: `/assets/logo-a1b2c3d4.png`).
- **특징**: 파일이 **코드에서 한 번만** 참조될 때, 그리고 **경로를 문자열로 직접 만들지 않고** import로 쓸 때 적합합니다.

### 2. public 디렉터리 (복사만, URL로 참조)

- **위치**: **반드시** 프로젝트 루트의 `public/` 폴더.
- **참조**: 코드에서 **import 하지 않고**, 문자열로 **절대 경로**만 사용합니다.  
  예: `'/sounds/bg.mp3'`, `\`/game05/asset/run/${i}.png\``
- **Vite 동작**:
  - `public/` 안의 내용은 **모듈 그래프에 넣지 않고**, `npm run build` 시 **그대로** `dist/` 루트로 복사합니다.
  - `public/sounds/foo.mp3` → 배포 시 `dist/sounds/foo.mp3` → 브라우저에서는 `/sounds/foo.mp3` 로 접근.
  - 파일 이름이나 경로를 Vite가 바꾸지 않으므로, **동적으로 경로를 조합**할 수 있습니다 (예: `asset/run/0.png` ~ `17.png`).
- **특징**: 개수가 많거나, 경로를 **런타임에 문자열로만** 만드는 경우, 또는 외부에서 **고정 URL**로 접근해야 하는 경우에 적합합니다.

정리하면, Vite는 **import 되면** “의존성 → 빌드 결과물로 변환”하고, **public에만 두면** “변환 없이 복사만 하고 URL은 그대로” 씁니다.

---

## 이 프로젝트에서의 구분

| 용도 | 폴더 | 참조 방식 | 예시 |
|------|------|-----------|------|
| import로 쓰는 리소스 | `resources/` | `import url from '...'` | 로고 이미지 (App, Game03 등) |
| URL 문자열로만 쓰는 리소스 | `public/` | `'/sounds/...'`, `\`/game05/...\`` | 사운드 파일, Game05 에셋 |

- **import 하면 `resources/`** 에 두고,  
- **URL로만 참조하면 `public/`** 에 두면 됩니다.
